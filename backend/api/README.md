# MoneyPulse API

FastAPI foundation for MoneyPulse orchestration, persistence, and REST contracts.

## Initial Endpoints

- `GET /health`
- `GET /accounts`
- `POST /accounts`
- `GET /transactions`
- `POST /transactions`
- `GET /goals`
- `POST /goals`
- `GET /today`
- `POST /before-you-buy`

## Goal

Keep the API thin while the Decision Engine remains deterministic, explainable, and well documented.
