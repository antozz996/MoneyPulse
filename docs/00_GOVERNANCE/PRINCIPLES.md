# Principles

Version: 1.0
Status: Approved
Owner: MoneyPulse Leadership

## Core Principles

1. Decisions over data.
2. Trust first.
3. Today first.
4. Simplicity wins.
5. One question per screen.
6. Explain every recommendation.
7. Mobile first.
8. Deterministic by default.
9. Depth before breadth.
10. Documentation before implementation.

## Practical Meaning

### Decisions over data

The product must lead with recommendations and consequences, not raw financial detail.

### Trust first

Every number shown to the user must be traceable to an input, rule, or assumption.

### Today first

The primary use case is a decision happening now, often on a phone, with limited time.

### Simplicity wins

If two designs solve the same problem, choose the one that is easier to understand and explain.

### One question per screen

Each screen should help the user answer a single practical question without forcing context switching.

### Explain every recommendation

MoneyPulse must never output a number without showing the main drivers behind it.

### Mobile first

The critical path must be fast, readable, and usable on a small screen before desktop enhancements are considered.

### Deterministic by default

The Decision Engine must produce the same output for the same input and model version.

### Depth before breadth

We go deep on daily affordability and purchase simulation before expanding into adjacent finance features.

### Documentation before implementation

Architecture, product intent, and AI behavior must be documented before code is treated as correct.
